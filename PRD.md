## 1. 产品概述

DeepRare Clinical 是一款面向**临床医师**的 AI 辅助罕见病诊断平台。核心能力分为两条诊断路径：

- **HPO 表型诊断**：医师输入自由文本病历，AI 通过多轮追问提取标准化 HPO 表型，再基于本体推理给出候选疾病排名及诊断依据。
- **VCF 基因诊断**：医师上传患者基因组变异文件（VCF），系统按 ACMG/AMP 指南对变异进行致病性分级并输出结构化报告。

两条路径共享同一套患者档案体系，支持多版本病史管理和多次诊断任务并发追踪。

### 2.1 主要用户旅程（HPO 诊断）

| 步骤 | 操作 | 系统响应 |
|------|------|----------|
| 1 | 点击导航「新建诊断」 | 进入新建诊断页，默认选中 HPO 表型诊断 |
| 2 | 搜索已有患者或留空 | 下拉展示患者列表；选中后自动预填基本信息 |
| 3 | 填写/补充病史文本 | 支持粘贴完整病历（主诉、现病史、既往史等） |
| 4 | 点击「发起 HPO 表型诊断」 | 创建新诊断任务，跳转至患者详情页·HPO 进行中视图 |
| 5 | 阶段 1：AI 追问症状 | AI 根据病历内容主动追问关键症状，医师选择或手动填写；追问结束后医师点击「确认，进入 HPO 确认」才进入阶段 2 |
| 6 | 阶段 2：HPO 清单确认 | 系统展示提取的 HPO 表型清单，医师可删减/手动添加；完成后点击「确认清单，开始推理」进入阶段 3 |
| 7 | 阶段 3：自动推理 | 后台多轮推理，实时显示推理日志和步骤进度；无需用户操作 |
| 8 | 诊断完成 | 患者状态更新为「待查看」；切换至「诊断完成」视图，展示候选疾病 Top 5 及详细报告 |

### 2.2 主要用户旅程（VCF 诊断）

| 步骤 | 操作 | 系统响应 |
|------|------|----------|
| 1 | 在新建诊断页切换至 VCF 诊断类型 | 表单增加 VCF 文件上传区 |
| 2 | 选择患者、填写病史 | 同 HPO 路径 |
| 3 | 上传 VCF 文件 | 支持 .vcf / .vcf.gz，单文件 ≤ 2GB，参考 GRCh38/GRCh37；上传过程中展示进度条，完成后显示文件名/大小 |
| 4 | 点击「发起 VCF 基因诊断」 | 创建诊断任务，跳转至患者详情页·VCF 进行中视图 |
| 5 | VCF 分析进行中 | 展示流水线步骤进度（质控 → 注释 → 过滤 → 变异解读 → 报告生成），无需用户操作 |
| 6 | 诊断完成 | 患者状态更新为「待查看」；切换至「诊断完成」视图，展示变异分组列表及 ACMG 详情 |

### 2.3 患者档案管理旅程

| 步骤 | 操作 | 系统响应 |
|------|------|----------|
| 1 | 进入患者列表页，点击患者 | 右侧展开患者档案面板 |
| 2 | 点击「编辑」 | 档案面板切换为编辑态 |
| 3 | 修改基本信息 / HPO 表型 / 病史 | 实时暂存未保存的修改 |
| 4 | 点击「保存」 | 弹出二次确认弹窗，选择仅保存 / 同时发起 HPO 诊断 / 同时发起 VCF 分析 |
| 5 | 保存成功 | 创建新病史版本（History v+1），关联本次修改 |
| 6 | 查看历史版本 | 点击「历史诊断」按钮弹出时间线弹层，按版本展示 HPO 表型与诊断任务 |

## 3. 页面详细设计

### 3.1 全局导航栏

#### 功能定位

全局顶部固定导航，提供页面跳转和全局操作入口。

#### 功能模块

| 区域 | 元素 | 行为 |
|------|------|------|
| 左侧 Logo 区 | DeepRare + CLINICAL 标签 | 点击无跳转，仅标识品牌 |
| 中间导航区 | 「新建诊断」「患者列表」两个按钮 | 点击切换页面；当前活跃项高亮 |
| 右侧操作区 | 设置图标按钮、用户头像 | 占位，暂无具体功能 |

#### 操作逻辑

- 导航高亮规则：用户位于新建诊断页时「新建诊断」高亮；位于患者列表或患者详情页时「患者列表」高亮。
- 当前所在页面在刷新后自动恢复；通过链接直接访问特定页面时，以链接中的目标页面为准。

---

### 3.2 新建诊断页

#### 功能定位

医师发起新一次诊断的入口，支持选择已有患者或创建新患者，填写诊断输入并提交。

#### 功能模块

##### 3.2.1 诊断类型切换

| 类型 | 标签 | 说明 |
|-----|------|------|
| HPO | HPO 表型诊断 | 基于症状文本，AI 推理候选疾病 |
| VCF | VCF 基因诊断 | 上传 .vcf/.vcf.gz 文件，进行变异致病性分析 |

- 默认选中 HPO 表型诊断。
- 切换诊断类型后，VCF 特有的文件上传区域显示或隐藏。
- **切换诊断类型不清空已填写内容**：患者选择、基本信息、病史文本在两种类型之间共享，切换后完整保留。VCF 类型在共用字段基础上额外展示文件上传区；切换回 HPO 类型后，之前选择的 VCF 文件需重新上传。

##### 3.2.2 患者选择（步骤 1）

**未选择患者时**

- 显示搜索输入框，提示文字：「按姓名或患者 ID 搜索已有患者...」
- 聚焦后展示搜索下拉，默认展示前 5 条患者记录。
- 搜索匹配逻辑：同时匹配姓名（精确）、患者 ID（大小写不敏感）、症状摘要（大小写不敏感）。
- 下拉列表每项展示：头像字符（姓名末字）、姓名、性别·年龄、症状摘要、患者 ID。
- 下拉末尾固定一条「未找到？提交时自动创建新患者」提示行。
- 点击搜索结果行 → 选中患者，下拉关闭，表单进入已选中态。

**已选中患者时**

- 显示已选患者卡片：头像字符、姓名、患者 ID·性别·年龄·HPO 数·VCF 数。
- 右侧显示「已选择 · 档案预填」标识和「切换」按钮。
- 点击「切换」→ 清空选择，回到搜索态。

**字段展示逻辑**

| 字段 | 显示位置 | 说明 |
|------|----------|------|
| 姓名末字 | 头像圆形 | 取名字最后一个字 |
| 姓名 | 主文字 | 粗体 |
| 性别 · 年龄 | 副文字 | 灰色小字 |
| 患者 ID | 右侧等宽字体 | `P-YYYY-NNNN` 格式 |
| HPO/VCF 数量 | 已选中态副文字 | `HPO n · VCF n` |

##### 3.2.3 患者信息 & 病史（步骤 2）

**只读模式**（已选已有患者时）

- 基本信息字段只读，无法手动修改。
- 表头提示「基本信息已预填，仅下方病史可追加」。
- 病史文本区：上方显示历史病史摘要（不可编辑），下方提供可编辑文本框用于追加新内容，提示文字为「追加此次就诊新症状、检查结果...」。

**可编辑模式**（未选患者，创建新患者时）

- 所有字段均可编辑。
- 病史文本框提示文字为「支持粘贴完整病历。可包含主诉、现病史、既往史、体格检查、辅助检查...」。

**表单字段规范**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本输入 | 是 | 提示：如：张三 |
| 性别 | 下拉选择 | 是 | 男 / 女 |
| 出生日期 | 文本输入 | 否 | 格式：YYYY-MM-DD |
| 民族 / 族裔 | 文本输入 | 否 | 提示：如：汉族 |
| 近亲婚配史 | 下拉选择 | 否 | 否 / 是 / 不详，默认「否」 |
| 家族史 | 多行文本 | 否 | 提示举例近亲婚配、不明原因肝病等 |
| 病史文本 | 多行文本 | 是 | 新建时必填，追加时选填（已有历史病史可不追加） |
| VCF 文件 | 文件上传区 | 是（VCF 类型） | 拖拽或点击，支持 .vcf/.vcf.gz，≤2GB，GRCh38/GRCh37 |

**VCF 文件上传状态**

| 状态 | 展示 |
|------|------|
| 待上传 | 虚线上传区，显示拖拽提示文字和上传图标 |
| 上传中 | 上传区内显示进度条 + 文件名 + 已上传/总大小（如 `1.2 GB / 2.0 GB`） |
| 上传成功 | 上传区变为文件卡片：文件名、文件大小、成功图标；右侧显示「重新上传」按钮 |
| 上传失败 | 错误提示（格式不支持 / 超过 2GB / 网络错误）+ 「重试」按钮 |

**表单校验规则**

- 点击提交按钮时触发校验；校验失败的字段在标签下方显示红色错误提示，输入框标红。
- 提交按钮始终保持可点击状态，在用户点击时校验并给出明确的错误提示，避免用户不清楚无法提交的原因。
- 必填字段为空时提示：「请填写 {字段名}」。
- VCF 类型下提交前未完成上传时提示：「请先上传 VCF 文件」。

##### 3.2.4 提交区（步骤 3）

- 左侧提示：「提交后 AI 推理将在 2–5 分钟内完成，可前往患者详情页查看进度。」
- 右侧仅保留「发起 HPO 表型诊断」或「发起 VCF 基因诊断」一个主要操作按钮，**无取消按钮**（医师可通过顶部导航直接离开，无需专门的取消操作）。
- 提交后跳转至该患者的 HPO 诊断进行中页面（HPO 路径）或 VCF 诊断进行中页面（VCF 路径）。

##### 3.2.5 从其他页面发起新诊断

当从患者列表页或患者详情页跳转至新建诊断页时，目标患者自动预选，表单以只读模式预填其基本信息，与手动搜索选中的效果一致。此机制使「为当前患者发起新诊断」的操作路径缩短为一步。

### 3.3 患者列表页

#### 功能定位

所有患者的总览入口，左侧为患者列表，右侧为当前选中患者的档案快览与诊断任务记录，支持在列表页直接编辑档案。

#### 布局结构

```
┌────────────────────────┬─────────────────────────────────────────┐
│       左侧患者列表      │            右侧患者档案面板              │
│  (固定宽度侧栏)         │  (无选中时显示空状态提示)                │
└────────────────────────┴─────────────────────────────────────────┘
```

#### 功能模块

##### 3.3.1 左侧患者列表

**列表头部**

| 元素 | 说明 |
|------|------|
| 标题「患者列表」 | 固定文字 |
| 患者总数标识 | 显示过滤后数量 |
| 隐私遮盖按钮（眼睛图标） | 切换后将所有患者姓名替换为「●●●」，头像替换为「●」，用于屏幕分享场景；开关状态在刷新后保持 |
| 搜索输入框 | 实时过滤，提示文字：「搜索姓名、ID、症状...」 |

**搜索过滤逻辑**

- 匹配患者姓名（精确包含）、患者 ID（大小写不敏感）、症状摘要（大小写不敏感）。
- 无结果时显示「暂无匹配患者」。

**患者行字段展示**

| 字段 | 位置 | 展示规则 |
|------|------|----------|
| 头像字符 | 左侧圆形 | 姓名末字；隐私模式显示「●」 |
| 姓名 | 主文字 | 隐私模式显示「●●●」 |
| 性别 · 年龄 | 姓名右侧 | 灰色小字，格式：`男 · 14岁` |
| 患者 ID | 副文字，等宽字体 | 格式：`P-YYYY-NNNN` |
| 末次诊断时间 | 副文字 | 格式：`末次诊断 YYYY-MM-DD` |
| 诊断状态 | 右侧 | 见状态定义；进行中状态显示动态进行中标识 |

**选中态**：点击患者行后该行高亮，右侧面板加载该患者档案。

**编辑态下的离开拦截**：若右侧面板当前处于编辑态（档案编辑未保存），以下任何离开操作均需弹出确认弹窗后才可执行：

| 触发场景 | 拦截方式 |
|----------|----------|
| 点击左侧其他患者行 | 弹窗确认后切换患者 |
| 点击顶部导航「新建诊断」 | 弹窗确认后跳转页面 |
| 点击顶部导航「患者列表」 | 弹窗确认后刷新列表（非编辑的列表态） |
| 点击右侧任务行（跳转患者详情页） | 弹窗确认后跳转 |

弹窗内容：
- 标题：「当前编辑未保存」
- 说明：「离开后，当前编辑的内容将丢失。是否继续？」
- 按钮：「返回继续编辑」（取消）/ 「放弃编辑并离开」（高危样式，丢弃未保存修改并执行原操作）

##### 3.3.2 右侧空状态提示

未选中任何患者时展示：

- 大号用户轮廓图标
- 标题：「从左侧选择患者」
- 副文字：「点击左侧列表中的患者，右侧将展示其档案信息与诊断任务记录。」
- 「新建诊断」按钮，点击跳转至新建诊断页。

##### 3.3.3 右侧患者档案面板

选中患者后，右侧面板分为三个区域：

**① 档案区（上左）**

*查看态字段*

| 字段 | 左侧信息列 | 说明 |
|------|-----------|------|
| 患者头像 + 姓名 | 区域头部 | 头像圆形（末字）+ 姓名（粗体）+ 性别（灰色） |
| 患者 ID | 区域头部副文字 | 等宽字体 |
| 诊断状态标识 | 区域头部 | |
| 性别 | 信息行 | |
| 出生日期 | 信息行 | 等宽字体 |
| 民族 | 信息行 | |
| 近亲婚配 | 信息行 | |
| 注册时间 | 信息行 | 等宽字体 |
| 最后就诊 | 信息行 | 等宽字体 |

*查看态 HPO 列（右列）*

- 标题「HPO 表型标签」
- 展示所有 HPO 标签（包含 HPO ID 和中文名称）。
- 无表型时显示「暂无 HPO 表型记录」。

*编辑态操作逻辑*

1. 点击头部「编辑」按钮 → 档案和病史区域均切换为编辑态，显示「编辑中」标识。
2. 编辑态分为左右两列：左列为基本信息表单，右列为 HPO 表型编辑器。
3. HPO 编辑：点击标签右侧 ✕ 删除；点击「添加表型」展开输入行（HP ID + 表型描述，支持回车确认）。
4. 点击「取消」→ 所有未保存修改丢弃，回到查看态。
5. 点击「保存」→ 弹出二次确认弹窗（见 3.3.4）。

**② 病史摘要区（下左）**

*查看态*

| 元素 | 说明 |
|------|------|
| 核心症状标签行 | 浅色背景标签，展示核心症状摘要 |
| 现病史文本 | 白色背景，保留原始换行，超出时可滚动查看 |
| 家族史块 | 独立区域，带「家族史」标签前缀 |

*编辑态*

| 字段 | 类型 | 说明 |
|------|------|------|
| 核心症状 | 单行文本 | 提示：如：构音障碍 · 震颤 · 肝酶升高 |
| 家族史 | 多行文本（可拉伸） | 提示：描述家族遗传病史、近亲婚配情况等... |

**③ 诊断任务区（右列）**

- 区域头部：标题「诊断任务」+ 任务数量标识 + 「新建」按钮（跳转至新建诊断页并**自动预选当前患者**）。
- 无任务时显示「暂无诊断记录 · 点击「新建」发起第一次诊断」。

*任务行字段*

| 字段 | 位置 | 说明 |
|------|------|------|
| 类型标签 | 左上 | `HPO`（蓝色）/ `VCF`（绿色） |
| 任务时间 | 中上 | 格式：`YYYY-MM-DD HH:mm` |
| 任务状态 | 右上 | 完成：绿色；进行中：动态进行中标识；失败：红色 |
| 任务结果摘要 | 主体文字 | 单行，超出省略 |
| 右箭头图标 | 右侧 | 视觉引导点击 |

### 4.5 实体关系说明

```
Patient 1 ──────────────── n History（病史版本）
   │              parent_version 字段形成版本链 (v1 → v2 → v3)
   │
   └──── 1 ───── n DiagnosisTask（诊断任务）
                     │
                     └──── n ──── 1 History（每次任务锁定具体版本）
```

- 一个患者拥有多个病史版本，版本间通过 `parent_version` 形成单向链表。
- 一个患者可发起多次诊断任务，可包含不同 `analysis_type`。
- 每次诊断任务在创建时锁定当前生效的病史版本（`history_id`），历史任务结果不受后续病史更新影响。

## 5. 状态流转与枚举定义

### 5.1 患者诊断状态（Patient.status）

| 状态值 | 标签 | 颜色 | 触发条件 |
|--------|------|------|----------|
| running | 诊断中 | 蓝色动态标识 | 患者存在**任意一个**诊断任务处于进行中状态 |
| review | 待查看 | 黄色 | 有诊断任务刚完成且该任务结果尚未被医师打开查看 |
| diagnosed | 诊断完成 | 绿色 | 所有任务均已完成且结果已被查看（无进行中 / 待查看状态任务） |
| failed | 诊断失败 | 红色 | 所有任务均已结束，最新一次为失败，且无进行中 / 待查看状态任务 |

**优先级规则**（存在多个任务并发时，取最高优先级）：

`进行中` > `待查看` > `诊断完成` > `诊断失败`

**待查看状态生命周期**：
- 触发：诊断任务由进行中变为完成时，患者状态自动置为待查看。
- 消除：医师打开该任务的详情结果页时，后端将该任务标记为"已查看"，若无其他待查看任务则患者状态变为诊断完成。

### 5.2 诊断任务状态（DiagnosisTask.status）

| 状态值 | 含义 |
|--------|------|
| running | 推理进行中 |
| done | 推理成功完成 |
| failed | 推理失败（文件错误或手动终止） |

### 5.3 ACMG 致病性分级

| 代码 | 含义 | 颜色 |
|------|------|------|
| P | Pathogenic 致病 | 深红 |
| LP | Likely Pathogenic 可能致病 | 红 |
| VUS | Variant of Uncertain Significance 意义未明 | 黄 |
| LB | Likely Benign 可能良性 | 灰绿 |
| B | Benign 良性 | 绿 |

### 5.4 遗传方式（inheritance）

| 代码 | 含义 |
|------|------|
| AR | 常染色体隐性 |
| AD | 常染色体显性 |
| XL | X 染色体连锁 |
| XR | X 染色体连锁隐性 |

## 6. 非功能性需求

### 6.1 性能

| 指标 | 要求 |
|------|------|
| 患者列表渲染 | ≤ 200ms（100 条以内） |
| 页面切换响应时间 | ≤ 100ms |
| HPO 推理等待提示 | 提交后立即跳转进行中视图，无需等待后台结果 |
| VCF 分析预期时长 | 页面提示 2–5 分钟，超时需展示失败状态 |

### 6.2 隐私保护

- 患者列表页提供「隐私遮盖」开关，一键隐藏所有患者姓名，用于屏幕展示或多人会议场景。
- 患者删除需二次确认，要求输入患者姓名后方可执行，且标注「此操作不可撤销」。
- **删除含进行中任务的患者**：若患者存在进行中状态的诊断任务，系统在删除时自动终止所有进行中任务（无需用户手动操作），随后执行删除。删除确认弹窗中需额外提示「该患者有正在进行中的诊断任务，删除后将一并终止」。

### 6.3 数据持久化

- 当前所在页面在刷新后自动恢复。
- 通过页面链接可直接访问特定诊断结果，支持链接分享。

### 6.4 可访问性

- 所有交互操作需有清晰的文字标识，支持辅助工具识别。
- 色彩不作为唯一信息载体（状态同时使用图标和文字）。

### 6.5 响应式

- 当前设计以桌面端（宽度 ≥ 1280px）为主；移动端暂不要求适配。
